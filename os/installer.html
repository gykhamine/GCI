GUIDE D'INSTALLATION DE LA PLATEFORME

01

Télécharger l'Image (.ISO)

La première phase requiert la récupération du fichier système brut. Il s'agit de l'image de disque complète de Gykhamine OS.

📥 Cliquer pour lancer le téléchargement de l'ISO
02

Graver l'image sur la Clé USB

Pour pouvoir démarrer votre ordinateur dessus, l'image téléchargée doit obligatoirement être gravée (écriture par blocs de bas niveau) sur une clé USB d'au moins 8 Go. Une simple copie de fichier ne fonctionnera pas.

Depuis Windows (Avec Rufus)

Utilisez l'utilitaire de gravure standard pour Windows :

🔗 Télécharger l'utilitaire Rufus
  • Insérez votre clé USB (attention : son contenu sera intégralement écrasé).
  • Lancez l'application Rufus.
  • Vérifiez que votre clé USB apparaît bien dans la section Périphérique.
  • Dans Type de démarrage, cliquez à droite sur le bouton SÉLECTION et sélectionnez le fichier ISO Gykhamine téléchargé.
  • Laissez toutes les autres options à leur valeur par défaut, puis cliquez sur Démarrer au bas de l'écran. Attendez la complétion de la barre de progression.

Depuis Linux (Via le Terminal)

Sous Linux, l'écriture s'effectue proprement en ligne de commande via l'utilitaire d'écriture brute dd :

  1. Ouvrez votre terminal et tapez la commande lsblk pour repérer le nœud de votre clé USB *(ex: sdb ou sdc)* selon sa taille.
  2. Démontez de force les volumes associés pour libérer le support :
    sudo umount /dev/sdb*
  3. Exécutez la commande de gravure en adaptant les chemins d'accès à votre environnement réel :
    sudo dd if=chemin_vers/gykhamine.iso of=/dev/sdb bs=4M status=progress oflag=sync
⚠️ Note de vigilance pour débutants : La gravure formate entièrement la clé USB cible. Prenez soin de cibler le bon disque pour éviter toute perte de données involontaire.
03

Configurer la Capsule Applicative

Dès que le processus de gravure s'achève, ouvrez le contenu de la clé USB à l'aide de votre explorateur de fichiers classique (ou gestionnaire de fichiers sous Linux). Vous êtes désormais à la racine de la partition Gykhamine Live (la partition EFI visible).

C'est à cet emplacement précis que vous déposez votre code applicatif et vos scripts. Vous devez y créer ou y modifier le dossier racine obligatoire nommé exactement Gykhamine, conçu pour accueillir vos couches applicatives de façon interchangeable.

Votre clé USB doit impérativement respecter la structure suivante :

📁 Racine_de_la_clé_USB (Partition EFI / Gykhamine Live)
├── 📁 EFI/ (Dossier système requis pour l'amorçage du PC)
├── 📁 live/ (Contient les composants vitaux : le noyau Linux, Dracut, et l'image Gykhamine brute)
└── 📁 Gykhamine/ (Votre dossier applicatif obligatoire — À créer ici)
    ├── 📁 1/gy.py (Scripts d'automatisation s'exécutant avec les privilèges d'administrateur SUDO)
    ├── 📁 2/gy.py (Scripts d'automatisation standards s'exécutant SANS privilège SUDO)
    └── 📁 my_project/ (L'arborescence complète de votre application web Django)
💡 Pour les débutants : Ne touchez jamais aux dossiers EFI/ et live/, ils servent à démarrer l'ordinateur. Tout votre travail se passe à l'intérieur du dossier Gykhamine/. Si vous devez changer de projet ou mettre à jour votre site web, il vous suffit de remplacer ce qui se trouve dans my_project/, 1/ et 2/.
04

Démarrage et Amorçage Automatique

Une fois l'arborescence validée, éjectez proprement le support USB.

  1. Insérez la clé USB préparée dans le port de la machine de destination (l'ordinateur doit être éteint).
  2. Mettez la machine sous tension et actionnez immédiatement et de façon répétée la touche d'accès au menu de boot de votre carte mère (généralement F12, F11, F8 ou Échap).
  3. Sélectionnez votre clé USB UEFI au sein de la liste proposée pour amorcer le système.
🚀 Automatisation Complète : Dès le boot initié, le noyau et dracut (situés dans le dossier live/) montent l'environnement immuable. Les routines système détectent le dossier Gykhamine/, exécutent séquentiellement vos automatisations système (dossier 1/), vos configurations utilisateur (dossier 2/), puis instancient le serveur de production de l'application Django présente dans my_project/ sans aucune action requise de votre part.
